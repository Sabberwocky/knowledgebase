# Contribution Guide

## How to Contribute to the Knowledgebase

### Adding New Content

1. **Choose the Right Category**
   - Review existing categories in `/categories/`
   - Create a new category if needed

2. **File Naming Conventions**
   - Use lowercase letters and hyphens: `my-topic.md`
   - Be descriptive but concise
   - Avoid spaces and special characters

3. **Content Format**
   - Use Markdown for all documentation
   - Include proper headings (H1, H2, H3)
   - Add metadata at the top of files:
   ```markdown
   ---
   title: "Topic Title"
   category: "category-name"
   tags: ["tag1", "tag2"]
   created: "2025-01-01"
   updated: "2025-01-02"
   ---
   ```

### Content Guidelines

- **Clarity**: Write clearly and concisely
- **Structure**: Use logical organization with headings
- **Examples**: Include practical examples when possible
- **Links**: Add cross-references to related topics
- **Images**: Place images in `/assets/` and reference them properly

### Review Process

1. Create a pull request
2. Ensure all content follows guidelines
3. Wait for review and feedback
4. Address any requested changes
5. Merge when approved

### Quality Standards

- Proofread for spelling and grammar
- Verify all links work correctly
- Test any code examples provided
- Ensure proper categorization
